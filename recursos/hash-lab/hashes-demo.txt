# ── bcrypt (SEGURO — diseñado para passwords) ─────────────────
# Incluye salt integrado | ~15,000 hashes/seg en GPU (cost 12)
# $2b$[cost]$[22 chars salt][31 chars hash]
$2b$12$EixZaYVK1fsbw1ZfbX3OXePaWxn96p36WQoeG6Lruj3vjPGga31lW    secret123
$2b$12$LQv3c1yqBWVHxkd0LHAkCOYz6TtxMQJqhN8/LewdBPj6ukx/LCoYe    mypassword
# NOTA: aunque el hash sea idéntico, dos usuarios con "admin" tendrán
# hashes distintos gracias al salt aleatorio automático.

# ── Argon2id (ÓPTIMO — PHC Winner 2015) ──────────────────────
# Memory-hard | resistente a GPU y ASIC | ~800 hashes/seg típico
# $argon2id$v=19$m=[mem_kib],t=[iters],p=[threads]$[salt]$[hash]
$argon2id$v=19$m=65536,t=3,p=4$c29tZXNhbHQ$RdescudvJCsgt3ub+b+dWRWJTmaaJObG   demo
$argon2id$v=19$m=65536,t=3,p=4$ZXhlbXBsb3NhbHQ$SV/I4n5qDQNb6jfNBmWBE3OYSQ6B   test

# =============================================================
# COMPARATIVA DE VELOCIDAD (GPU RTX 3090, 2024)
#   MD5     → 10,000,000,000 hashes/seg  → INSEGURO
#   SHA1    →  3,300,000,000 hashes/seg  → INSEGURO
#   SHA256  →  1,200,000,000 hashes/seg  → MAL USO para passwords
#   bcrypt  →         15,000 hashes/seg  → SEGURO
#   Argon2  →            800 hashes/seg  → ÓPTIMO
#
# Conclusión: para almacenar passwords usar SIEMPRE bcrypt o Argon2.
