# Contributing Guide

Gracias por tu interés en contribuir a este proyecto.

## Objetivo del proyecto

Este repositorio es una plataforma educativa orientada a:

* Criptografía aplicada
* Ciberseguridad
* SOC (Security Operations Center)
* Threat Intelligence
* DFIR / Forensia Digital
* Detection Engineering
* Análisis de tráfico y telemetría
* Protocolos y seguridad de red
* Casos prácticos y laboratorios

El objetivo es construir material técnico, práctico y didáctico que ayude a comprender cómo se aplican estos conceptos en entornos reales.

---

# Tipos de contribuciones aceptadas

Se aceptan contribuciones relacionadas con:

* nuevos laboratorios
* mejoras de documentación
* análisis técnicos
* ejemplos prácticos
* visualizaciones educativas
* mejoras UI/UX
* optimización responsive
* correcciones técnicas
* contenido orientado a SOC/DFIR
* análisis de protocolos
* ejemplos de detección
* ejercicios y casos de estudio

---

# Antes de contribuir

Por favor:

1. Revisar la estructura existente del proyecto
2. Mantener coherencia visual y de navegación
3. Reutilizar componentes ya existentes cuando sea posible
4. Evitar duplicación innecesaria
5. Mantener compatibilidad con GitHub Pages

---

# Reglas técnicas

## Frontend

El proyecto utiliza:

* HTML
* CSS
* JavaScript

Preferir soluciones simples, legibles y fácilmente mantenibles.

---

# Estilo visual

Mantener:

* estética consistente
* diseño responsive
* navegación clara
* estructura modular
* accesibilidad básica

Evitar:

* cambios globales innecesarios
* romper navegación existente
* introducir complejidad innecesaria

---

# Contenido

El contenido debe:

* estar en español
* ser técnicamente correcto
* mantener enfoque educativo
* incluir contexto práctico cuando sea posible
* priorizar claridad y comprensión

---

# Buenas prácticas

Preferir:

* cambios incrementales
* commits descriptivos
* código comentado
* ejemplos claros
* documentación breve y útil

---

# Ejemplos de commits

```bash
feat(tls): add JA3S analysis section

fix(css): improve responsive layout in labs

docs(soc): add detection workflow examples
```

---

# Seguridad y responsabilidad

No subir:

* credenciales reales
* secretos
* tokens
* información sensible
* infraestructura productiva
* material ilegal

Utilizar:

* datos simulados
* ejemplos controlados
* entornos de laboratorio

---

# Filosofía del proyecto

La idea del proyecto es conectar teoría y práctica mediante:

* laboratorios interactivos
* análisis técnicos
* ejemplos reales
* visualizaciones didácticas
* ejercicios aplicados
* escenarios orientados a defensa y análisis

buscando que los conceptos puedan entenderse no solo desde lo académico, sino también desde su aplicación operativa en ciberseguridad.
