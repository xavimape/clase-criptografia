Wireshark — Análisis TLS

Cómo capturar, filtrar y extraer información JA3 de tráfico TLS usando Wireshark y tshark desde la perspectiva del analista SOC.

TLS Lab → 🔑 JA3 Core 🦈 Wireshark 🐷 Snort 📊 QRadar 🛡️ FortiGate

Filtros de display de Wireshark para aislar tráfico TLS y enfocarse en los Client Hello que producen las huellas JA3. Son distintos de los filtros de captura (BPF).

Filtros esenciales — Client Hello

WIRESHARK DISPLAY FILTER Todos los TLS Client Hello 
tls.handshake.type == 1

El tipo de mensaje 1 es siempre un Client Hello. Es el punto de partida para cualquier análisis JA3 — todos los paquetes que aparecen con este filtro son candidatos a generar una huella JA3.

WIRESHARK DISPLAY FILTER Client Hello con versión TLS específica 
/* TLS 1.2 Client Hello (0x0303) */
tls.handshake.type == 1 && tls.handshake.version == 0x0303

/* TLS 1.0 Client Hello — inusual en 2024, sospechoso */
tls.handshake.type == 1 && tls.handshake.version == 0x0301

/* Clientes que NO ofrecen TLS 1.3 — potencialmente legacy/malware */
tls.handshake.type == 1 && !(tls.handshake.extensions.supported_version == 0x0304)
Insight SOC: En 2024, cualquier cliente que negocie TLS 1.0 o 1.1 es sospechoso. Muchos malwares usan versiones antiguas para compatibilidad con C2 servers viejos o mal configurados.

Filtrar por Cipher Suites

WIRESHARK DISPLAY FILTER Cipher suites débiles o sospechosas 
/* Cipher NULL — sin cifrado, tráfico en claro */
tls.handshake.ciphersuite == 0x0000

/* DES / RC4 — algoritmos rotos desde hace años */
tls.handshake.ciphersuite == 0x0009 || tls.handshake.ciphersuite == 0x0005

/* Sólo 1-4 cipher suites en la lista — típico de tools simples o malware */
tls.handshake.type == 1 && tls.handshake.ciphersuites.count <= 4

/* Client Hello con TLS_RSA_WITH_NULL_MD5 — nunca debería verse */
tls.handshake.ciphersuite == 0x0001

Filtrar por Extensiones TLS

WIRESHARK DISPLAY FILTER Detección de clientes sin extensiones modernas 
/* Client Hello sin SNI (Extension 0) — raro en browsers legítimos */
tls.handshake.type == 1 && !(tls.handshake.extensions.type == 0x0000)

/* Sin ALPN — inusual para browsers pero normal en algunos tools */
tls.handshake.type == 1 && !(tls.handshake.extensions.type == 0x0010)

/* Sin session ticket — indica cliente sin persistencia de sesión */
tls.handshake.type == 1 && !(tls.handshake.extensions.type == 0x0023)

/* Buscar SNI específico (dominio sospechoso) */
tls.handshake.extensions_server_name == "evil-c2.example.com"

Filtros combinados para investigación

WIRESHARK DISPLAY FILTER Combinaciones útiles para análisis SOC 
/* Todos los Client Hello desde una IP sospechosa */
tls.handshake.type == 1 && ip.src == 192.168.1.105

/* Client Hello hacia puerto no estándar (no 443) — posible C2 */
tls.handshake.type == 1 && tcp.dstport != 443 && tcp.dstport != 8443

/* Client Hello hacia rango de IPs de C2 conocido */
tls.handshake.type == 1 && (ip.dst >= 185.220.0.0 && ip.dst <= 185.220.255.255)

/* Ver flujo completo de una sesión TLS */
tls.handshake.type == 1 && ip.src == 192.168.1.105
/* Luego: clic derecho → Follow → TLS Stream */

Wireshark 3.x+ puede mostrar el hash JA3 directamente en el panel de detalle del paquete sin plugins adicionales. También existe el plugin Lua para versiones antiguas.

Método 1 — JA3 nativo en Wireshark 3.3+

Desde Wireshark 3.3, el disector TLS calcula JA3 automáticamente. Para verlo:

  1. Abrí el PCAP con un Client Hello TLS
  2. Seleccioná un paquete con tls.handshake.type == 1
  3. En el panel de detalle, expandí: Transport Layer Security → TLSv1.3 Record Layer → Handshake Protocol: Client Hello → JA3 Fullstring / JA3
  4. El campo JA3 muestra el hash MD5 de 32 chars
WIRESHARK FIELD Filtro de display para el hash JA3 
/* Ver solo paquetes con hash JA3 calculado */
tls.handshake.ja3

/* Filtrar por hash JA3 específico — Metasploit */
tls.handshake.ja3 == "e7d705a3286e19ea42f587b344ee6865"

/* Buscar Cobalt Strike default */
tls.handshake.ja3 == "de350869b8c85de67a350c8d186f11e6"

/* Buscar cualquier hash que NO sea browser conocido */
tls.handshake.ja3 && !(
  tls.handshake.ja3 == "773906b0efdefa24a7f2b8eb6985bf37" ||
  tls.handshake.ja3 == "53b7c45e93f5d2f5765a1cf5e61c1e26" ||
  tls.handshake.ja3 == "07f3d598e197f6255e17ef9b0c6cd22a"
)

tshark es la versión CLI de Wireshark. Ideal para análisis automatizado, scripting y procesamiento de PCAPs grandes en servidores sin GUI. Fundamental para pipelines SOC.

Extracción básica de campos JA3

BASH · tshark Extraer todos los hashes JA3 de un PCAP 
# Extraer: timestamp, src_ip, dst_ip, dst_port, ja3_hash, sni
tshark -r captura.pcap \
  -Y "tls.handshake.type == 1" \
  -T fields \
  -e frame.time_epoch \
  -e ip.src \
  -e ip.dst \
  -e tcp.dstport \
  -e tls.handshake.ja3 \
  -e tls.handshake.extensions_server_name \
  -E header=y \
  -E separator=, \
  -E quote=d \
  > ja3_output.csv

# Resultado ejemplo:
# "frame.time_epoch","ip.src","ip.dst","tcp.dstport","tls.handshake.ja3","tls..."
# "1704067200.000","192.168.1.105","185.220.101.4","443","e7d705a3286e19ea42f587b344ee6865","update.evil.com"

  done

Script Python para análisis masivo de JA3

PYTHON Análisis de PCAP con pyshark + correlación de threat intel 
#!/usr/bin/env python3
"""
ja3_analyzer.py — Análisis de huellas JA3 en PCAPs
Uso: python3 ja3_analyzer.py captura.pcap
"""
import subprocess
import csv
import sys
from collections import Counter

# ── Base de firmas conocidas (en producción: cargar desde feed) ──
KNOWN_MALICIOUS = {
    "e7d705a3286e19ea42f587b344ee6865": "Metasploit Meterpreter",
    "de350869b8c85de67a350c8d186f11e6": "Cobalt Strike (pre-4.x)",
    "8bab54f22de07d88e90f5e8e5edb5a69": "Emotet C2",
    "19e29534fd49dd27d09234e7b95b86c1": "Dridex / BitPaymer",
}

def extract_ja3(pcap_file):
    """Usa tshark para extraer campos JA3 del PCAP."""
    cmd = [
        "tshark", "-r", pcap_file,
        "-Y", "tls.handshake.type == 1",
        "-T", "fields",
        "-e", "ip.src",
        "-e", "ip.dst",
        "-e", "tcp.dstport",
        "-e", "tls.handshake.ja3",
        "-e", "tls.handshake.extensions_server_name",
        "-E", "separator=|",
    ]
    result = subprocess.run(cmd, capture_output=True, text=True)
    return result.stdout.strip().split("\n") if result.stdout else []

def analyze(pcap_file):
    records = extract_ja3(pcap_file)
    hashes  = []
    alerts  = []

    for line in records:
        parts = line.split("|")
        if len(parts) < 4 or not parts[3]:
            continue

        src, dst, port, ja3, sni = parts + [""] * (5 - len(parts))
        hashes.append(ja3)

        if ja3 in KNOWN_MALICIOUS:
            alerts.append({
                "hash": ja3, "name": KNOWN_MALICIOUS[ja3],
                "src": src, "dst": dst, "sni": sni or "(sin SNI)"
            })

    print(f"\n=== JA3 Analysis: {pcap_file} ===")
    print(f"Total Client Hello packets: {len(records)}")
    print(f"Unique JA3 hashes: {len(set(hashes))}")

    print("\n── Top 5 hashes más frecuentes ──")
    for h, count in Counter(hashes).most_common(5):
        tag = f"  ⚠ {KNOWN_MALICIOUS[h]}" if h in KNOWN_MALICIOUS else ""
        print(f"  {h}  ×{count}{tag}")

    if alerts:
        print(f"\n🔴 ALERTAS ({len(alerts)} detecciones):")
        for a in alerts:
            print(f"  [{a['name']}] {a['src']} → {a['dst']} (SNI: {a['sni']})")
    else:
        print("\n✅ Sin hashes maliciosos conocidos detectados.")

if __name__ == "__main__":
    if len(sys.argv) < 2:
        print("Uso: python3 ja3_analyzer.py ")
        sys.exit(1)
    analyze(sys.argv[1])

Flujo de análisis forense con Wireshark para investigar una alerta de JA3 malicioso. Pasos ordenados por prioridad de tiempo — primero contención, luego análisis profundo.

Checklist de análisis TLS con Wireshark

🎯
Aislar el tráfico del host afectado

Aplicar filtro: ip.src == [IP_SOSPECHOSA] || ip.dst == [IP_SOSPECHOSA]
Verificar volumen de tráfico, cantidad de sesiones TLS y rangos horarios. Un patrón de beacon (tráfico periódico cada N segundos) es indicativo de C2.

🔍
Analizar el Client Hello sospechoso

Filtro: ip.src == [IP] && tls.handshake.type == 1
Expandir en el panel de detalle: ¿cuántas cipher suites ofrece? ¿Tiene SNI? ¿Qué extensiones? Un Client Hello con pocas ciphers y sin SNI es un indicador fuerte de herramienta automatizada.

📋
Registrar el hash JA3 y el SNI

Copiar el hash JA3 del panel de detalle (campo tls.handshake.ja3). Copiar el SNI (tls.handshake.extensions_server_name) — el dominio contactado puede estar en Alexa Top 1M (DGA que mimetiza dominios legítimos) o ser un DGA puro.

🔗
Seguir el flujo TLS completo

Clic derecho en el paquete → Follow → TLS Stream
Si tenés la clave privada del servidor (entorno de lab o corporate proxy), podés descifrar. Sin clave: observar tamaños de Application Data para inferir tipo de payload. Patrones regulares de 300-800 bytes cada 30-60 seg → beacon C2.

📊
Estadísticas de conversaciones

Statistics → Conversations → TCP
Ver duración, bytes transferidos y cantidad de paquetes por sesión. C2 channels suelen tener sesiones largas con bajo volumen de datos (poll de comandos) o sesiones cortas muy frecuentes (beaconing con jitter).

💾
Exportar evidencia

File → Export Specified Packets → guardar el PCAP filtrado al flujo sospechoso.
Hashear el PCAP con SHA256 para cadena de custodia: sha256sum evidencia.pcap
Documentar: timestamp, interfaz de captura, filtros aplicados, hash del archivo.
