FortiGate / FortiAnalyzer — JA3 y SSL Inspection

Configuración de SSL/TLS inspection en FortiGate, logging de huellas JA3, y análisis con FortiAnalyzer para detección de C2 en tráfico cifrado.

TLS Lab → 🔑 JA3 Core 🦈 Wireshark 🐷 Snort 📊 QRadar 🛡️ FortiGate

FortiOS 6.4+ incluye logging nativo de huellas JA3 y JA3S como parte del motor de SSL Inspection. Para obtener datos JA3, no es necesario hacer deep inspection (descifrado) — JA3 se extrae del Client Hello sin certificado de CA intermedio.

Modos de SSL Inspection en FortiGate

Modo ¿Descifra? ¿JA3 disponible? Caso de uso
Certificate Inspection No ✓ Sí (Client Hello) Productivo — bajo impacto, JA3 del cliente
Full SSL Inspection (MITM) Sí ✓ Sí + JA3S AV, IPS, DLP sobre HTTPS — requiere CA en clientes
No Inspection No ✗ No Solo para tráfico excluido por compliance
Recomendación SOC: Para detección JA3 sin impacto en la experiencia del usuario, usar Certificate Inspection en todas las políticas por defecto. Solo aplicar Full SSL Inspection en segmentos críticos (servidores, VIPs) donde se necesite inspección de contenido.

Aplicar en: Firewall Policy → Security Profiles → SSL/SSH Inspection

Configurar SSL Inspection Profile — CLI FortiOS

FORTIOS CLI config ssl-ssh-profile — configuración completa 
# Crear perfil de Certificate Inspection con logging JA3
config firewall ssl-ssh-profile
    edit "SOC-Certificate-Inspection"
        set comment "Certificate inspection con JA3 logging para SOC"

        # Modo: solo inspección de certificados, sin descifrado
        set ssl-negotiation-log enable       # Loguear handshake TLS
        set ssl-server-cert-log enable       # Loguear cert del servidor

        # Protocolo HTTPS
        config https
            set ports 443 8443 4443          # Agregar puertos adicionales si C2 usa otros
            set status certificate-inspection
        end

        # TLS en puertos no estándar — sospechosos de C2
        config ssl
            set inspect-all enable           # Inspeccionar cualquier TLS, no solo HTTPS
        end

        # Habilitar logging de JA3 (requiere FortiOS 6.4.x+)
        set ja3-enabled enable              # ← Campo clave para huellas JA3

        # Rechazar conexiones con certificados autofirmados/expirados
        # (C2 servers frecuentemente usan self-signed certs)
        set invalid-server-cert block       # O "allow" + log si no querés bloquear

        # Versiones TLS obsoletas — bloquear o alertar
        set min-allowed-ssl-version tls-1.2  # No permitir TLS 1.0/1.1
    next
end

# Aplicar el perfil a una política de firewall existente
config firewall policy
    edit 10    # ID de la política LAN→WAN
        set ssl-ssh-profile "SOC-Certificate-Inspection"
        set logtraffic all                  # Loguear todo el tráfico
        set logtraffic-start enable         # Log al inicio de la sesión
    next
end

Una vez habilitado el SSL Inspection Profile con JA3, FortiGate incluye los campos ja3 y ja3s en los logs UTM. Estos logs pueden verse en el GUI, exportarse a FortiAnalyzer o enviarse a un SIEM externo vía Syslog.

Configurar envío de logs a SIEM (Syslog)

FORTIOS CLI Syslog hacia QRadar / Splunk / SIEM externo 
# Configurar syslog con los campos relevantes para JA3
config log syslogd setting
    set status enable
    set server 10.0.0.50         # IP del SIEM / Syslog collector
    set port 514
    set mode udp                  # UDP o TCP dependiendo del SIEM
    set facility local7           # Facility syslog
    set format rfc5424            # o "default" para formato FortiGate
    set priority default          # Loguear todos los niveles
end

# Especificar qué tipos de log enviar al syslog
config log syslogd filter
    set severity information      # Incluir todos los logs desde "information" hacia arriba
    set forward-traffic enable    # Logs de sesiones (incluyen JA3)
    set utm-virus enable
    set utm-ips enable
    set utm-app-ctrl enable
    set utm-ssl enable            # ← CRÍTICO: logs UTM SSL contienen JA3
end

# Verificar que los logs llegan al SIEM:
# En QRadar: Log Activity → buscar "source: FortiGate" y campo "ja3"
# En Splunk: index=firewall sourcetype=fortigate | stats count by ja3

FortiAnalyzer centraliza los logs de todos los FortiGates de la organización y provee una interfaz para queries SQL-like (FAZ Query Language) y dashboards de seguridad. Los campos JA3 del SSL log son consultables directamente.

Reports personalizados con JA3

FORTIANALYZER SQL Report Builder — Dataset SQL para huellas JA3 
# Reports → Report Definitions → New → Dataset → SQL

-- Dataset: Top JA3 hashes (último día)
SELECT
    ja3,
    ja3s,
    sni,
    COUNT(DISTINCT srcip)  AS hosts_origen,
    COUNT(*)               AS total_conexiones,
    MIN(itime)             AS primera_vez,
    MAX(itime)             AS ultima_vez
FROM $log-ssl-all           -- vista de logs SSL de todos los devices
WHERE
    ja3 IS NOT NULL
    AND ja3 != ''
    AND itime >= NOW() - 86400  -- últimas 24 horas (en segundos para FAZ)
GROUP BY ja3, ja3s, sni
ORDER BY total_conexiones DESC
LIMIT 20

-- Dataset: Hosts con JA3 de malware conocido
SELECT
    srcip,
    dstip,
    dstport,
    sni,
    ja3,
    tlsver,
    itime AS timestamp,
    CASE ja3
        WHEN 'e7d705a3286e19ea42f587b344ee6865' THEN 'Metasploit Meterpreter'
        WHEN 'de350869b8c85de67a350c8d186f11e6' THEN 'Cobalt Strike'
        WHEN '8bab54f22de07d88e90f5e8e5edb5a69' THEN 'Emotet C2'
        WHEN '19e29534fd49dd27d09234e7b95b86c1' THEN 'Dridex'
        ELSE 'Other Malicious'
    END AS threat_name
FROM $log-ssl-all
WHERE
    ja3 IN (
        'e7d705a3286e19ea42f587b344ee6865',
        'de350869b8c85de67a350c8d186f11e6',
        '8bab54f22de07d88e90f5e8e5edb5a69',
        '19e29534fd49dd27d09234e7b95b86c1'
    )
    AND itime >= NOW() - 86400
ORDER BY itime DESC

Application Control de FortiGate puede bloquear tráfico basado en la firma de la aplicación detectada a nivel TLS. En FortiOS 7.x, se puede crear firmas de aplicación personalizadas usando campos JA3 para bloqueo proactivo.

Custom Application Signature basada en JA3

FORTIOS CLI Firma de aplicación custom para Metasploit JA3 
# Crear una firma de aplicación personalizada para Metasploit
# basada en la huella JA3

config application custom
    edit "Metasploit-Meterpreter-TLS"
        set comment "Detección por JA3 hash de Meterpreter"
        set protocol SSL                # Protocolo de detección
        set technology Browser-Based    # Categoría
        set category 21                 # Misc (ajustar según taxonomía)
        set risk 5                      # Riesgo máximo

        # Condición de match: contenido del Client Hello
        # El JA3 hash no se puede buscar directamente como contenido
        # en App Control — se usa la combinación de versión + cipher bytes
        # Alternativamente, usar IPS custom signature:
    next
end

# ── MEJOR OPCIÓN: IPS Custom Signature ────────────────────────────────────
# IPS puede inspeccionar el payload TLS y hacer content matching
config ips custom
    edit "JA3-Metasploit-Meterpreter"
        set comment "Bloquear TLS Client Hello de Meterpreter por cipher list"
        set severity critical
        set location client
        set action block
        set protocol TCP
        set dst-port 443
        # Patrón en el Client Hello de Meterpreter:
        # Versión TLS 1.0 (0x0301) + cipher suites específicos
        set signature
            "F-SBID( --protocol TCP; --dst_port 443;
             --content \"|16 03 01|\"; --within 3;
             --content \"|00 04 05 0A 09 64|\"; --byte_jump 0,36;
             --classtype trojan-activity; )"
    next
end

Integración con FortiGuard Threat Intelligence

# FortiGuard incluye feeds de IPs y dominios C2 conocidos
# Cuando hay un hit de JA3 sospechoso + IP en el feed de C2 → bloqueo automático

config webfilter profile

                # Botnet C2 (categoría 26) — bloquear

                end
                # Malware sites (categoría 82) — bloquear
                edit 2
                    set category 82
                    set action block
                end
                # Phishing (categoría 83) — bloquear
                edit 3
                    set category 83
                    set action block
                end
            end
        end
    next
end

# Habilitar IPS signatures de Botnet (complementa JA3):
config ips sensor
    edit "SOC-IPS-Profile"
        config entries
            # Todas las firmas de Botnet
            edit 1
                set rule Botnet.*
                set action block
                set log enable
            end
        end
    next
end

Checklist de hardening TLS en FortiGate

  • Habilitar SSL Inspection Profile con ja3-enabled en todas las políticas

    Sin esto, no hay visibilidad JA3. Mínimo: Certificate Inspection en políticas LAN→WAN.

  • Bloquear versiones TLS < 1.2

    config ssl-ssh-profile → set min-allowed-ssl-version tls-1.2

  • Configurar envío de logs UTM SSL a FortiAnalyzer o SIEM

    Sin centralización de logs, la visibilidad JA3 es solo local y difícil de correlacionar.

  • Crear Alert Policy en FortiAnalyzer para hashes JA3 críticos

    Notificación automática al equipo SOC cuando aparece un hash de malware conocido.

  • Habilitar IPS con firmas de Botnet y C2

    Complementa la detección JA3 con firmas de payload. Doble capa de detección.

  • Revisar logs JA3 semanalmente y actualizar baseline

    Los clientes TLS cambian con actualizaciones. Actualizar el baseline mensualmente para reducir FP.

  • Configurar inspección TLS en puertos no estándar

    C2 usa puertos 4444, 5555, 8888, 9999. Agregar al SSL profile y al Application Control.
