JA3 Fingerprinting Lab

Laboratorio interactivo para entender cómo se generan las huellas TLS JA3, cómo identificar clientes maliciosos y cómo integrar esta técnica en el flujo SOC.

TLS Lab → 🔑 JA3 Core 🦈 Wireshark 🐷 Snort 📊 QRadar 🛡️ FortiGate

JA3 (John Althouse, Jeff Atkinson, Josh Atkins — Salesforce, 2017) extrae cinco campos del mensaje TLS Client Hello y calcula su hash MD5. El resultado es una huella de 32 chars que identifica la implementación TLS del cliente — independientemente de la IP o el dominio contactado.

Fórmula JA3

JA3 construye un string con los 5 campos, separando los valores de cada lista con - y los campos entre sí con ,. Luego calcula el MD5 del resultado:

FÓRMULA Concatenación de campos 
JA3_STRING = SSLVersion,Ciphers,Extensions,EllipticCurves,ECPointFormats

Ejemplo (Chrome 96):
  771,4865-4866-4867-49195-49199-49196-49200-52393-52392-49171-49172-156-157-47-53,\
  0-23-65281-10-11-35-16-5-13-18-51-45-43-27-17513,\
  29-23-24,\
  0

JA3_HASH = MD5(JA3_STRING)
         = cd08e31494f9531f560d64c695473da9  ← huella final de 32 chars
¿Por qué MD5? JA3 usa MD5 por velocidad de cálculo en línea de red — no por seguridad criptográfica. Con ~10 mil millones de hashes/segundo en GPU, el análisis en tiempo real de tráfico no introduce latencia notable.

Valores GREASE — RFC 8701

Los browsers modernos insertan valores GREASE (Generate Random Extensions And Sustain Extensibility) para garantizar que los servidores ignoren valores TLS desconocidos. JA3 los filtra antes de concatenar para mantener huellas consistentes entre sesiones del mismo cliente.

Valores GREASE (decimal) Hex ¿Dónde aparece?
2570, 6682, 10794, 14906, 19018, 23130, 27242, 31354 0x0a0a…0x7a7a Ciphers, Extensions, Curves
35466, 39578, 43690, 47802, 51914, 56026, 60138, 64250 0x8a8a…0xfafa Ciphers, Extensions, Curves

Insight SOC: El mismo hash JA3 aparece independientemente del dominio contactado. Un Metasploit Meterpreter contactando google.com o evil.onion produce la misma huella. Esto hace que JA3 sea efectivo para detección en escenarios de DGA o fast-flux.

Base de firmas JA3. Hashes documentados en threat intel público. Buscá un hash para verificar si aparece en la base, o filtrá por categoría. Importante: los hashes varían con versiones de software — la ausencia no implica benignidad.

Flujo SOC con JA3. Desde la alerta inicial hasta el cierre del incidente. Los pasos asumen que ya tenés visibilidad TLS (proxy, IDS, EDR o tap de red con exportación a SIEM).

Flujo de detección y respuesta

1
🔔 Alerta SIEM / IDS — hash JA3 sospechoso

El SIEM correlaciona un hash JA3 contra la lista de firmas conocidas. Puede ser una regla Snort, una alerta de QRadar o un indicador de FortiGate. La alerta incluye: src_ip, dst_ip, dst_port, ja3_hash, timestamp.

2
🔍 Enriquecimiento del indicador

Verificar el hash en feeds externos: sslbl.abuse.ch, ja3er.com, VirusTotal (Network tab). Correlacionar con el historial del host origen: ¿es un servidor, una estación de trabajo, un dispositivo IoT? ¿Cuándo fue la primera vez que viste este JA3 en tu red?

3
📦 Captura de evidencia

Si confirmás que es malicioso: capturar el PCAP completo del flujo, exportar logs del SIEM del host afectado (Event IDs 3, 7045, 4688 en Windows), tomar snapshot de memoria si el host está activo. El PCAP puede revelar el payload C2.

4
🔗 Análisis de pivote

Buscar el mismo JA3 hash en toda la red: ¿hay otros hosts con la misma huella contactando el mismo C2? El mismo JA3 en múltiples fuentes sugiere infección masiva o movimiento lateral. Buscar también por dst_ip y certificate_hash.

5
🛑 Contención y bloqueo

Bloquear la IP/dominio de destino en el firewall y proxy. Si el malware usa DGA, el bloqueo por IP es insuficiente — evaluar aislamiento del host. Agregar el JA3 hash a la lista de bloqueo en Snort/Suricata, QRadar y FortiGate.

6
📝 Documentación y lecciones aprendidas

Actualizar el playbook con el nuevo JA3. Agregar el hash a la watchlist del SIEM. Reportar el indicador a plataformas de threat intel compartido (MISP, ISACs). Evaluar si la detección fue oportuna o tardía y ajustar el umbral de alerta.

Herramientas del ecosistema SOC para JA3

Cada herramienta tiene su propio laboratorio con configuración detallada:

🦈
Wireshark

Análisis de PCAPs, filtros de display TLS, disección de Client Hello y extracción manual de campos JA3.
🐷
Snort / Suricata

Reglas de detección en tiempo real. Alertas por hash JA3 conocido y patrones de cipher suites anómalos.
📊
IBM QRadar

Custom Properties para extraer JA3, AQL queries de correlación y offenses automáticos por amenaza.
🛡️
FortiGate / FortiAnalyzer

SSL/TLS inspection, JA3 logging nativo en FortiOS 6.4+, queries en FortiAnalyzer para correlación.
